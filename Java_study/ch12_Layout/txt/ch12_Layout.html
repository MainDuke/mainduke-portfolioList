LayoutManager : 컨포넌트 배치 관리자

 

 

1. FlowLayout :  Panel 계열의 디폴트 이다 ( Panel , Applet )

2. BorderLayout : Window 계열의 디폴트 이다 ( Frame , Dialog, FileDialog)

3. GridLayout : 동일한 격자 모양으로 배치

4. GridBagLayout : 컴포넌트를 다양한 크기로 배치

5. CardLayout : 카드식으로 컴포넌트 배치

6. null  : 프로그래머가 직접 컴포넌트 크기와 위치를 지정해 준다

 

 

FlowLayout

-   Panel과 Applet 의 디폴트 Layout 이다

-   FlowLayout 특징

   컨테이너크기를 사용자가 변경하였을때 , 컴포넌트 크기는 변화지 않고 상대적인 위치만 변한다

 

- 만약 Panel 계열이 아닌 컨테이너에 FlowLayout 을 적용하려면

컨테이너 클래스의 setLayout()메소드에 FlowLayout 형 오브젝트를 넘겨 주어야한다

 

Frame f;

f.setLayout(new FlowLayout());

 

- FlowLayout 생성자

 1.  public FlowLayout()

         가운데 정렬 , 컴포넌트 간의 수평/수직 거리는 5픽셀인 FlowLayout를 생성한다

 

 2.  public FlowLayout(int align, int hgap, int vgap)

           FlowLayout.LEFT : 왼쪽정렬

           FlowLayout.RIGHT : 오른쪽정렬

           FlowLayout.CENTER : 가운데정렬

 

          hgap :   컴포넌트의 수평 거리 ( 픽셀 단위 이다 )

          vgap :   컴포넌트의 수직 거리

 

BorderLayout

1. BorderLayout은 Window 계열 Container에 기본적으로 적용되는 LayoutManager이다

 

2. 당연히 Window클래스의 하위 클래스인 Frame클래스의 경우에도 특별히 LayoutManager를 변경하지 않는한 BorderLayout이 적용된다

 

3. BorderLayout특징 : Container의 크기를 사용자가 변경시키면

        Component의 상대적인 위치는 변하지 않고 크기만 변한다

 

4. 따라서 BorderLayout이 적용된 Container의 경우는

Component를 Frame의 어느 위치에  붙일것인지 add()메소드에 다음과 같이 지정한다

c1.add(c2,n);

c1  : BorderLayout이 적용될 Container오브젝트

c2  : Container붙일 Component오브젝트

n  : 붙일 위치를 나타내는 상수

 

      n

5. 만약 Window계열이 아닌 Container를  BorderLayout를 적용하려면

Container클래스 setLayout()메소드에 BorderLayout형 오브젝트를 넘겨준다

 

p 핸들 변수가 가리키는 Panel의 LayoutManager를 BoerderLayout으로 하려면

p.setLayout(new BorderLayout());

 

6. BorderLayout()생성자 : java.awt.BorderLayout클래스 소속

   원형 : public BerderLayout()

        * 컴포넌트 사이의 간격이 없는 BoerderLayout생성

   원형 : public BorderLayout(int hgap, int vgap)

        * 컴포넌트 사이에 지정된 간격이 있는 BorderLayout생성

 

 

GridLayout

1. GridLayout은 Component를 동일한 크기의 격자(Grid)나누어 Container 에 표시

 

2. GridLayout을 적용하려면

Container클래스 메소드인 setLayout()메소드에 GridLayout형 오브젝트를 넘겨준다.

 

예) f 핸들 변수가 가리키는Frame의 LayoutManager를 GridLayout하려면

   f.setLayout(new GridLayout(3,2));

                              줄,칸

   f.setLayout(new GridLayout(0,2));

0은 add() 메소드를 호출한 횟수에 따라 변할 수 있다.

add()가 6번 호줄되었다면  3행 2열로된다

 

3. 특징 : Container의 크기를 사용자가 변경시켰을 때 그 위에 붙어있는

      각 Component의 상대적위치는 변하지 않고 크기만 변함

 

4. GridLayout()생성자

원형 : public GridLayout()

     * 한 행에서 컴포넌트만큼 열이 있는 GridLayout을 실행

원형 : public GridLayout(int rows, int cols)

    * 지정한 행, 열 수를 가진 GridLayout생성

원형 :public GridLayout(int rows, int cols, int hgap, int vgap)

    * 행수, 열수 , 수평간격, 수직 간격

throws … illegalArgumentException  ..행,열이 지정되지 않았을 경우
